# Support

Kawakiri is research software under active development.

## Usage questions

Read the [README](README.rst), [usage guide](docs/usage.md), and
[troubleshooting section](README.rst#troubleshooting) first. If the question remains,
open a GitHub issue and describe the expected outcome.

## Bug reports

Include the following information in a GitHub issue:

- Kawakiri version or commit hash;
- Python, operating-system, and ClickHouse versions;
- the command that was executed;
- the complete error message;
- a minimal, non-sensitive CSV example when possible;
- whether the problem occurs with `code/data`.

## Feature requests

Describe the research or data-engineering need, the expected behavior, and a small
example. Proposals that preserve deterministic, explainable inference are preferred.

## Scope of support

Community support is provided on a best-effort basis. A `VALID` certification means
that the implemented rules passed for the supplied data; it is not a guarantee of
business correctness, production suitability, or professional data-governance advice.
